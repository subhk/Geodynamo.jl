Getting Started

Basic Thermal Convection

Start with a simple thermal convection simulation:

using Geodynamo

function basic_thermal_convection()
    # 1. Create configuration
    config = create_optimized_config(32, 32, nlat=64, nlon=128)
    
    # 2. Set up radial domain (Earth-like)
    domain = create_radial_domain(0.35, 1.0, 64)
    
    # 3. Create temperature field
    temp_field = create_shtns_temperature_field(Float64, config, domain)
    
    # 4. Simple boundaries: hot bottom, cold top
    temp_boundaries = create_hybrid_temperature_boundaries(
        (:uniform, 4000.0),    # Hot CMB (4000 K)
        (:uniform, 300.0),     # Cool surface (300 K)
        config
    )
    
    # 5. Apply boundaries
    apply_netcdf_temperature_boundaries!(temp_field, temp_boundaries)
    
    println("✓ Basic thermal convection setup complete!")
end

basic_thermal_convection()

Programmatic Boundary Patterns

Create realistic boundary conditions using built-in patterns:

# Hot plume at CMB
temp_boundaries = create_hybrid_temperature_boundaries(
    (:plume, 4200.0, Dict(
        "center_theta" => π/3,    # Plume location (60° from north)
        "center_phi" => 0.0,      # Prime meridian
        "width" => π/6            # 30° plume width
    )),
    (:uniform, 300.0),            # Uniform cool surface
    config
)

# Spherical harmonic patterns
y11_boundaries = create_hybrid_temperature_boundaries(
    (:y11, 4000.0, Dict("amplitude" => 200.0)),  # Y₁₁ + perturbation
    (:uniform, 300.0),                           # Uniform surface
    config
)

# Available patterns: :uniform, :y11, :y20, :plume, :hemisphere, 
#                    :dipole, :quadrupole, :checkerboard, :custom

:uniform

Constant value across boundary

:y11

Y₁₁ spherical harmonic pattern

:plume

Gaussian plume structure

:custom

Your own mathematical function

Using NetCDF Data Files

Load boundary conditions from observational or model data:

# 1. Create sample NetCDF files (one-time setup)
include("examples/create_sample_netcdf_boundaries.jl")

# 2. Load temperature boundaries from NetCDF files
temp_boundaries = load_temperature_boundaries("cmb_temp.nc", "surface_temp.nc")

# 3. Load compositional boundaries
comp_boundaries = load_composition_boundaries("cmb_comp.nc", "surface_comp.nc")

# 4. Apply to fields
apply_netcdf_temperature_boundaries!(temp_field, temp_boundaries)
apply_netcdf_composition_boundaries!(comp_field, comp_boundaries)

# 5. Inspect loaded data
print_boundary_info(temp_boundaries)
stats = get_boundary_statistics(temp_boundaries.inner_boundary)
println("CMB temperature range: [$(stats["min"]), $(stats["max"])] K")

NetCDF File Format

Your NetCDF files should include:

  • Coordinates: theta (colatitude) and phi (longitude)
  • Data variable: temperature, composition, etc.
  • Optional: time dimension for time-dependent boundaries
  • Metadata: units, descriptions, and attributes

Hybrid Approaches

Mix NetCDF data with programmatic patterns for maximum flexibility:

# NetCDF inner boundary + programmatic outer boundary
temp_boundaries = create_hybrid_temperature_boundaries(
    "realistic_cmb.nc",       # Complex CMB from data file
    (:uniform, 300.0),        # Simple uniform surface
    config
)

# Programmatic inner + NetCDF outer
temp_boundaries = create_hybrid_temperature_boundaries(
    (:plume, 4200.0, Dict("width" => π/6)),  # Analytical plume pattern
    "earth_surface_temps.nc",                # Real surface temperature data
    config
)

# Custom function example
my_pattern(theta, phi) = sin(3*theta) * cos(2*phi) + 0.5*cos(theta)^2

custom_boundaries = create_hybrid_temperature_boundaries(
    (:custom, 4000.0, Dict("function" => my_pattern)),  # Your function
    (:uniform, 300.0),                                  # Simple outer
    config
)

Why Hybrid?

  • Flexibility: Use data where available, analytical where not
  • Performance: Simple boundaries where physics allows
  • Testing: Compare complex vs. simple boundary effects
  • Development: Gradually add complexity to boundaries
